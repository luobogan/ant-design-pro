## 📋 概述

本文档基于 Sword 项目的权限管理实现（参考 `docs/Sword 按钮权限 API 分析.md`），分析在 ant-design-pro 项目中实现动态按钮权限和用户信息获取功能的最佳位置。

**分析目标：**
- 确定动态按钮权限获取功能的最佳实现位置
- 确定用户信息获取功能的最佳实现位置
- 评估 `app.tsx` vs `usePermission.ts` 两种方案的优劣
- 提供清晰的实施建议

// 获取按钮权限
export function getButtons(): any[] {
  const buttonsStr = localStorage.getItem(BUTTONS_KEY);
  if (!buttonsStr) return [];
  try {
    return JSON.parse(buttonsStr);
  } catch {
    return [];
  }
}

// 获取指定代码的按钮权限
export function getButton(code: string): any[] {
  const buttons = getButtons();
  const data = buttons.filter((d: any) => d.code === code);
  return data.length === 0 ? [] : data[0].buttons;
}

// 设置按钮权限
export function setButtons(buttons: any[]): void {
  localStorage.removeItem(BUTTONS_KEY);
  localStorage.setItem(BUTTONS_KEY, JSON.stringify(buttons));
}
```

- 工具函数形式
- 基于 localStorage 存储
- 与 Sword 项目保持一致

## 📊 实现方案对比

### 方案 A：在 `app.tsx` 中实现

#### 实现方式

```typescript
// src/app.tsx
export async function getInitialState(): Promise<{
  settings?: Partial<LayoutSettings>;
  currentUser?: API.CurrentUser;
  loading?: boolean;
  fetchUserInfo?: () => Promise<API.CurrentUser | undefined>;
  fetchButtons?: () => Promise<void>;
  buttons?: any[];
}> {
  const fetchUserInfo = async () => {
    const res = await queryCurrentUser();
    return res.data;
  };
  
  const fetchButtons = async () => {
    try {
      const response = await dynamicButtons();
      setButtons(response.data || []);
    } catch (error) {
      console.error('获取按钮权限失败:', error);
    }
  };
  
  if (location.pathname !== loginPath) {
    const currentUser = await fetchUserInfo();
    // 并行获取用户信息和按钮权限
    await Promise.all([
      fetchUserInfo(),
      fetchButtons()
    ]);
    
    return {
      fetchUserInfo,
      fetchButtons,
      currentUser,
      settings: defaultSettings,
    };
  }
  
  return {
    fetchUserInfo,
    fetchButtons,
    settings: defaultSettings,
  };
}
```

#### 优点 ✅

1. **统一初始化入口**
   - 所有全局数据在 `getInitialState` 中统一获取
   - 符合 Umi 4 的最佳实践
   - 与现有的 `fetchUserInfo` 逻辑一致

2. **应用启动时自动加载**
   - 在应用渲染前完成数据获取
   - 避免组件渲染时的权限闪烁
   - 提供一致的用户体验

3. **全局可访问**
   - 通过 `initialState` 在所有组件中可访问
   - 可在 `layout`、`menu` 等配置中使用
   - 支持在运行时更新

4. **SSR 友好**
   - 支持服务端渲染场景
   - 数据可以在服务端预取

5. **与现有架构对齐**
   - 与 Sword 项目的 `menu.js` 模型类似
   - 在应用启动时获取权限数据
   - 符合已有的开发模式

#### 缺点 ❌

1. **增加初始化时间**
   - 额外的 API 请求会延长启动时间
   - 用户需要等待更长时间才能看到界面

2. **职责混合**
   - `app.tsx` 已经包含路由、布局、菜单等多个职责
   - 再加入按钮权限会使文件更加臃肿
   - 不符合单一职责原则

3. **灵活性较低**
   - 每次应用启动都会获取，无法按需加载
   - 难以实现局部刷新或增量更新

4. **测试困难**
   - `app.tsx` 的测试相对复杂
   - 难以单独测试权限获取逻辑

### 方案 B：在 `usePermission.ts` 中实现

#### 实现方式

```typescript
// src/hooks/usePermission.ts
import { useEffect, useState, useCallback } from 'react';
import { dynamicButtons } from '@/services/system/menu';
import { getButtons, setButtons, getButton } from '@/utils/authority';
import { useModel } from '@umijs/max';

export const usePermission = () => {
  const [loading, setLoading] = useState(false);
  const [buttons, setButtonsState] = useState<any[]>([]);
  const { initialState } = useModel('@@initialState');
  const currentUser = initialState?.currentUser;
  
  // 检查是否为管理员
  const isAdmin = currentUser?.roles?.includes('admin') || false;
  
  // 获取按钮权限
  const fetchButtons = useCallback(async () => {
    // 先从缓存获取
    const cached = getButtons();
    if (cached && cached.length > 0) {
      setButtonsState(cached);
      return;
    }
    
    // 缓存不存在则从 API 获取
    setLoading(true);
    try {
      const response = await dynamicButtons();
      const buttonsData = response.data || [];
      setButtons(buttonsData); // 存储到 localStorage
      setButtonsState(buttonsData);
    } catch (error) {
      console.error('获取按钮权限失败:', error);
    } finally {
      setLoading(false);
    }
  }, []);
  
  // 检查权限
  const hasPermission = useCallback((permission?: string): boolean => {
    if (!permission) return true;
    if (isAdmin) return true;
    
    // 从全局状态或 localStorage 获取权限
    const allButtons = getButtons();
    // 检查是否有该权限
    // 实际逻辑需要根据项目结构调整
    return true;
  }, [isAdmin]);
  
  // 过滤按钮
  const filterButtons = useCallback((buttonList: any[]) => {
    if (!buttonList || buttonList.length === 0) return [];
    return buttonList.filter((btn) => hasPermission(btn.permission || btn.code));
  }, [hasPermission]);
  
  // 获取指定页面的按钮
  const getPageButtons = useCallback((code: string) => {
    return getButton(code);
  }, []);
  
  return {
    loading,
    buttons,
    isAdmin,
    hasPermission,
    filterButtons,
    getPageButtons,
    fetchButtons,
  };
};

export default usePermission;
```

#### 在组件中使用

```typescript
// src/pages/System/User/User.tsx
import { usePermission } from '@/hooks/usePermission';

const UserPage: React.FC = () => {
  const { hasPermission, filterButtons, getPageButtons, fetchButtons } = usePermission();
  const [buttons, setButtons] = useState<any[]>([]);
  
  useEffect(() => {
    // 获取按钮权限
    const loadButtons = async () => {
      await fetchButtons(); // 如果未加载则获取
      const pageButtons = getPageButtons('system.user');
      setButtons(pageButtons);
    };
    loadButtons();
  }, []);
  
  return (
    <PageContainer
      extra={buttons.filter(btn => btn.action === 1).map(btn => (
        <Button key={btn.code}>{btn.name}</Button>
      ))}
    >
      <ProTable
        columns={columns}
        toolBarRender={() => [
          hasPermission('user:add') && <Button>新增</Button>,
        ].filter(Boolean)}
      />
    </PageContainer>
  );

```

#### 优点 ✅

1. **关注点分离**
   - 权限逻辑集中在专门的 Hook 中
   - `app.tsx` 专注于应用配置
   - 符合单一职责原则

2. **按需加载**
   - 只在需要权限的组件中获取
   - 可以减少不必要的 API 请求
   - 支持懒加载和代码分割

3. **高度可复用**
   - 可在任意组件中使用
   - 支持多个组件独立调用
   - 易于单元测试

4. **灵活性高**
   - 可以实现缓存策略
   - 支持增量更新
   - 可以根据不同场景定制逻辑

5. **符合 React 最佳实践**
   - 使用 Hook 模式
   - 函数式组件风格
   - 易于理解和维护

#### 缺点 ❌

1. **可能需要多次 API 调用**
   - 如果多个组件同时调用，可能重复请求
   - 需要实现缓存机制避免重复请求

2. **权限闪烁风险**
   - 组件首次渲染时权限数据可能未就绪
   - 需要处理 loading 状态
   - 可能导致 UI 闪烁

3. **全局状态管理复杂**
   - 需要与 `initialState` 协调
   - 可能需要额外的状态管理方案

4. **与 Sword 项目不一致**
   - Sword 在 model 中统一获取
   - 当前方案更加分散

   */
  const filterButtons = (buttonList: ButtonConfig[]) => {
    if (!buttonList || buttonList.length === 0) return [];
    return buttonList.filter((btn) => hasPermission(btn.permission || btn.code));
  };
  
  /**
   * 获取指定页面的按钮权限
   * @param code 页面代码（如：'system.user'）
   * @returns 按钮权限数组
   */
  const getPageButtons = (code: string) => {
    return getButton(code);
  };
  
  /**
   * 检查页面是否有指定按钮
   * @param pageCode 页面代码
   * @param buttonCode 按钮代码
   * @returns 是否有权限
   */
  const hasPageButton = (pageCode: string, buttonCode: string): boolean => {
    const pageButtons = getPageButtons(pageCode);
    return checkButton(pageButtons, buttonCode);
  };
  
  return {
    hasPermission,
    filterButtons,
    getPageButtons,
    hasPageButton,
    isAdmin,
    buttons,
    loading: false, // 数据已在 app.tsx 中获取
  };
};

export default usePermission;
```

## 📝 总结

### 核心建议

1. **数据获取放在 `app.tsx`**
   - 利用 `getInitialState` 在应用启动时获取
   - 与用户信息一起并行获取，提高性能
   - 存储到 `initialState.buttons` 和 localStorage

2. **权限判断放在 `usePermission.ts`**
   - 使用 Hook 模式，便于在组件中使用
   - 从 `initialState` 或 localStorage 读取数据
   - 提供 `hasPermission`、`filterButtons`、`getPageButtons` 等方法

3. **数据存储使用 `authority.ts`**
   - 保持与 Sword 项目的一致性
   - 提供统一的 localStorage 操作接口
   - 支持缓存和离线场景

### 优势总结

✅ **性能最优**：只获取一次，全局共享  
✅ **职责清晰**：每个文件专注于单一职责  
✅ **易于维护**：符合 React 和 Umi 的最佳实践  
✅ **用户体验好**：无权限闪烁，加载流畅  
✅ **高度可复用**：Hook 模式便于在任意组件中使用  
✅ **与现有架构对齐**：与 `fetchUserInfo` 模式一致
